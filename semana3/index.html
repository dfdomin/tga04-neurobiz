📚 Marco Teórico

1. Estructura condicional simple 🟢 VIGENTE

La estructura SI-FSI permite ejecutar instrucciones solo cuando una condición es verdadera. Si la condición resulta falsa, el algoritmo sigue su camino sin ejecutar ese bloque.

🚦

Analogía de vida real: SI el semáforo está en rojo ENTONCES detenerse FSI. Si está en verde, no hay instrucción especial: simplemente continúas.

💼

Analogía empresarial: SI el monto de la compra supera $1.000.000 ENTONCES aplicar 10% de descuento FSI. La acción ocurre solo cuando la condición se cumple.

2. Sintaxis Cairó 🟢 VIGENTE

En la notación del curso, la estructura se escribe como SI condición ENTONCES ... FSI. Es una manera clara de visualizar dónde comienza y dónde termina el bloque condicional.

📖

¿Qué significan las siglas SI-FSI?
SI = "Si" (empieza la condición a evaluar)  ·  ENTONCES = "entonces, ejecuta esto"  ·  FSI = "Fin Si" (cierra el bloque — indica hasta dónde llega el SI)

🐍 Equivalente en Python: if condición: — en Python no existe FSI porque el cierre del bloque se indica con indentación (sangría de 4 espacios). El concepto es idéntico; solo cambia la forma de escribirlo.

🧱 Molde general — SI-FSI
SI condicion ENTONCES
  { instrucciones que solo ocurren si es VERDADERA }
FSI

3. Diagrama de flujo del SI-FSI 🟢 VIGENTE

El rombo de decisión siempre tiene dos salidas: una para Sí y otra para No. En el SI-FSI simple, la rama verdadera ejecuta acciones; la rama falsa continúa sin hacer nada adicional.

ElementoQué representaLectura correcta
RomboPregunta lógica¿La tasa es menor que 70?
Salida “Sí”Bloque ENTONCESMostrar alerta
Salida “No”Continuación normalNo se muestra alerta

4. Condiciones compuestas con Y / O 🟢 VIGENTE

Cuando el negocio necesita verificar dos cosas al mismo tiempo, se usan condiciones compuestas. Y exige que ambas sean verdaderas; O permite que una sola baste.

CondiciónInterpretaciónResultado esperado
tasa_retencion < 70.0 Y sesiones_fallidas > 50Alerta dobleMás severa
tipo_terapia = "cognitiva" O tipo_terapia = "ocupacional"Pertenece a uno de dos gruposBasta una coincidencia

5. ¿Cuándo usar SI-FSI y cuándo no? 🟡 EVOLUCIONADO

Usa SI-FSI cuando solo quieres reaccionar a un caso verdadero. Si necesitas un camino alterno cuando la condición es falsa, en la Semana 4 aprenderás SI-SINO-FSI. En Python moderno, la idea sigue vigente con if: y operadores and/or.

⚠️

Pista importante: si el estudiante siente que “falta algo” cuando la condición es falsa, probablemente ya está necesitando un SINO. Eso se verá la próxima semana.

6. Errores frecuentes al leer un SI-FSI 🟢 VIGENTE

ErrorQué cree el estudianteCorrección docente

📝 Pseudocódigo — Alertas con SI-FSI

📝 Pseudocódigo — AlertaRetencionNeuroBiz
ALGORITMO AlertaRetencionNeuroBiz
  VARIABLES
    tasa_retencion: REAL
    META_RETENCION: REAL
  INICIO
    META_RETENCION ← 70.0
    LEER tasa_retencion
    SI tasa_retencion < META_RETENCION ENTONCES
      ESCRIBIR "⚠️ ALERTA: Tasa de retención crítica"
      ESCRIBIR "Tasa actual: ", tasa_retencion, "%"
      ESCRIBIR "Meta mínima: ", META_RETENCION, "%"
    FSI
    ESCRIBIR "Análisis completado."
  FIN
📝 Pseudocódigo — AlertaDobleNeuroBiz
ALGORITMO AlertaDobleNeuroBiz
  VARIABLES
    tasa_retencion: REAL
    sesiones_fallidas: ENTERO
  INICIO
    LEER tasa_retencion
    LEER sesiones_fallidas
    SI tasa_retencion < 70.0 Y sesiones_fallidas > 50 ENTONCES
      ESCRIBIR "🚨 ALERTA CRÍTICA: Intervención urgente requerida"
    FSI
  FIN

🧩

Regla de oro: primero calcula la condición, después decide si el bloque ENTONCES entra o se omite. Ese orden lógico evita que el estudiante “invente” resultados.

Resumen operacional: entrada → evaluación lógica → posible acción → continuidad del algoritmo.

Si no puedes explicar verbalmente la condición, todavía no está lista para codificarse.

Practicar la lectura en voz alta fortalece la precisión lógica.
