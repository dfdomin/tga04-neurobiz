1. Variable y tipo de dato 🟢 VIGENTE

Una variable es un espacio con nombre donde el algoritmo guarda información temporal. El tipo de dato define qué clase de valor puede almacenar esa variable. En negocios, esto evita errores de captura, cálculos erróneos y reportes inconsistentes.

🗄️

Analogía del cajón: si el cajón dice ENTERO, allí van números enteros. Si dice CADENA, allí va texto. Si guardas "cognitiva" en un cajón pensado para números, el sistema se rompe igual que una fórmula de Excel mal construida.

🏢

Puente empresarial: cuando una organización define mal sus tipos de datos, aparecen errores de integración, reportes duplicados y tableros que no cuadran. Esta semana formaliza las variables que en la Semana 1 solo estaban nombradas.

3. Reglas para nombrar variables 🟢 VIGENTE

  • Sin espacios: usar tasa_retencion, no tasa retencion.
  • Sin tildes ni caracteres especiales: numero_sesiones, no número_sesiones.
  • No empezar con número: sesiones2026 sí, 2026sesiones no.
  • Usar nombres que expliquen el dato: meta_retencion es mejor que x.
⚠️

Error clásico: mezclar tipos en una fórmula. Si concatenas un número con una cadena sin conversión, herramientas como Excel devuelven #VALOR!. En programación sucede lo mismo: el algoritmo necesita saber exactamente qué tipo está usando.

6. Operadores lógicos y expresiones compuestas 🟢 VIGENTE

Los operadores lógicos combinan condiciones. Se usan cuando el negocio no pregunta una sola cosa, sino varias al mismo tiempo.

OperadorLecturaEjemplo¿Cuándo es verdadero?
YANDtasa_retencion >= 70 Y sesiones_fallidas < 120Solo si ambas condiciones se cumplen
OORtipo_terapia = "cognitiva" O tipo_terapia = "ocupacional"Si al menos una condición se cumple
NONOTNO es_activoInvierte el valor lógico
🧠

Expresión compuesta clave: la fórmula principal del caso es tasa_retencion ← (sesiones_asistidas / total_sesiones) * 100. Observa que combina suma, división y multiplicación dentro de una sola expresión formal.

7. Notación de asignación de Cairó 🟡 EVOLUCIONADO

En este curso escribimos ← para asignar un valor a una variable. En Python moderno se usa =. La idea es la misma: guardar un resultado en un nombre. La notación cambia; el concepto sigue vivo.

📝 Pseudocódigo — Variables y Operaciones en NeuroBiz

El siguiente algoritmo formaliza los indicadores de NeuroBiz usando los tipos de dato del curso. Léelo de arriba hacia abajo y verifica dónde se declara, se calcula y se imprime cada valor.

📝 Pseudocódigo — CalcularKPIsNeuroBiz
ALGORITMO CalcularKPIsNeuroBiz
  VARIABLES
    sesiones_asistidas: ENTERO
    sesiones_fallidas: ENTERO
    total_sesiones: ENTERO
    tasa_retencion: REAL
    nombre_terapeuta: CADENA
    es_activo: LÓGICO
  INICIO
    LEER sesiones_asistidas
    LEER sesiones_fallidas
    LEER nombre_terapeuta
    total_sesiones ← sesiones_asistidas + sesiones_fallidas
    tasa_retencion ← (sesiones_asistidas / total_sesiones) * 100
    ESCRIBIR "Terapeuta: ", nombre_terapeuta
    ESCRIBIR "Tasa retención: ", tasa_retencion, "%"
    es_activo ← sesiones_asistidas > 0
    ESCRIBIR "Terapeuta activo: ", es_activo
  FIN
