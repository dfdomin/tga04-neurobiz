¿Qué es una prueba de escritorio? 🟢 VIGENTE

Es la simulación manual de un algoritmo con datos conocidos. En lugar de ejecutar código, seguimos cada instrucción, anotamos el cambio de las variables y validamos si el resultado coincide con lo esperado.

👨‍🍳

Analogía de vida real: como un chef que prueba el plato paso a paso: revisa la sal, corrige, vuelve a probar y solo sirve cuando la receta está controlada.

🏢

Analogía empresarial: una auditoría de procesos revisa transacción por transacción para confirmar que el procedimiento se cumplió. La tabla de trazado es la auditoría del algoritmo.

Tabla de trazado y casos de prueba 🟢 VIGENTE

Las columnas representan variables; las filas, pasos de ejecución. Para que una prueba sea útil, debe cubrir datos normales, valores límite y datos erróneos.

Algoritmo NeuroBiz a trazar 🟡 EVOLUCIONADO

Este algoritmo integra entrada, acumulación, condicionales anidadas y ciclo PARA. Es el cierre natural de RA3 antes de pasar a Python.

ALGORITMO PruebaNeuroBiz_RA3
  VARIABLES
    n: ENTERO
    i: ENTERO
    tasa: REAL
    verde: ENTERO
    amarillo: ENTERO
    rojo: ENTERO
    suma: REAL
  INICIO
    verde ← 0
    amarillo ← 0
    rojo ← 0
    suma ← 0
    n ← 3
    PARA i ← 1 HASTA n FPA
      LEER tasa
      suma ← suma + tasa
      SI tasa >= 80 ENTONCES
        verde ← verde + 1
      SINO
        SI tasa >= 50 ENTONCES
          amarillo ← amarillo + 1
        SINO
          rojo ← rojo + 1
        FSI
      FSI
    FPA
    ESCRIBIR suma / n
    ESCRIBIR verde, amarillo, rojo
  FIN

Las funciones en Python se definen con def. Son la traducción directa de los subalgoritmos y módulos del pseudocódigo. Abre Google Colab.

🐍

Subalgoritmo → función Python:
El pseudocódigo SUBALGORITMO calcularTasa(asistidas, programadas) se convierte en:
def calcular_tasa(asistidas, programadas):
El RETORNAR del pseudocódigo es return en Python.

🐍 Python — Funciones NeuroBiz (equivale a subalgoritmos)
# NeuroBiz — Módulo de análisis de sesiones con funciones

def calcular_tasa(asistidas, programadas):
    """Calcula y retorna la tasa de asistencia."""
    if programadas > 0:
        return asistidas / programadas * 100
    return 0.0

def clasificar_sesion(tasa):
    """Retorna el color de estado según la tasa."""
    if tasa >= 80:
        return "Verde ✅"
    elif tasa >= 60:
        return "Amarillo ⚠️"
    else:
        return "Rojo 🔴"

def reporte_ejecutivo(sesiones):
    """Imprime el resumen ejecutivo de NeuroBiz."""
    print("\n=== REPORTE EJECUTIVO NEUROBIZ ===")
    totales = {"Verde ✅": 0, "Amarillo ⚠️": 0, "Rojo 🔴": 0}
    for s in sesiones:
        tasa = calcular_tasa(s['asistidas'], s['programadas'])
        estado = clasificar_sesion(tasa)
        totales[estado] += 1
        print(f"  {s['id']:<4} Tasa: {tasa:.1f}% → {estado}")
    print(f"\nResumen: {totales}")

# Programa principal
sesiones_demo = [
    {'id': 'S01', 'asistidas': 42, 'programadas': 50},
    {'id': 'S02', 'asistidas': 31, 'programadas': 50},
    {'id': 'S03', 'asistidas': 48, 'programadas': 50},
]
reporte_ejecutivo(sesiones_demo)
🧠

¿Por qué funciones? Cada función resuelve UN problema. Si mañana cambia la regla de clasificación (ej: 70% en vez de 80%), solo modificas clasificar_sesion() — el resto del código no cambia. Esto es modularidad.
