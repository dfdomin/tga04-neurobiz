📚 Marco Teórico

1. Dato vs. Información 🟢 VIGENTE

Un dato es un hecho crudo, sin contexto: "847", "2026-01-15", "Ana García". Una información es un dato procesado que responde una pregunta de negocio: "La tasa de retención de NeuroBiz en enero fue del 78%". (Cairó Battistutti & Guardati, 2022)

💡

Analogía: Los ingredientes de una receta son datos. El plato terminado y servido es la información: ha sido procesada, tiene contexto y responde una necesidad.

🏛️

Harvard CS50 Business: "The difference between data and information is the same as the difference between ingredients and a meal — one requires transformation to have value." — David Malan, CS50 Business Professionals, Harvard University.

2. Software y Sistemas de Información 🟢 VIGENTE

El software es el conjunto de instrucciones que le indica al computador qué hacer con los datos. Un sistema de información integra personas, procesos, datos y tecnología para apoyar la toma de decisiones. (Joyanes Aguilar, 2008)

En NeuroBiz, el sistema que vamos a construir algorítmicamente tomará datos de las sesiones y los convertirá en indicadores de gestión que la dirección pueda interpretar.

🧠

Relevancia empresarial (WEF, 2025): El Foro Económico Mundial identifica el pensamiento analítico como la competencia #1 para 2025–2030. Saber distinguir dato de información, y diseñar el proceso de transformación entre ellos, es la base de toda carrera en gestión empresarial moderna.

3. ¿Qué es un Algoritmo? 🟢 VIGENTE

Un algoritmo es una secuencia finita, precisa y efectiva de pasos para resolver un problema. Las tres propiedades son fundamentales (Cairó & Guardati, 2022):

PropiedadSignificadoEjemplo en NeuroBiz
FinitoSiempre terminaEl proceso de calcular la tasa de retención tiene un principio y un fin claros
PrecisoSin ambigüedad; cada paso es único"Dividir sesiones asistidas entre total de sesiones" — no hay otra interpretación
EfectivoCada paso es realizableUn computador puede ejecutar cada instrucción matemática del algoritmo
💡

Analogía empresarial: El SOP (Standard Operating Procedure / Procedimiento Operativo Estándar) de incorporación de un empleado nuevo es un algoritmo de negocio: tiene pasos definidos, en orden, que siempre terminan con el empleado incorporado. Cada paso es preciso: "Entregar credencial de acceso en RR.HH. → Asignar puesto de trabajo → Instalar computador → Iniciar inducción de 8 horas."

4. Representación de Algoritmos 🟡 EVOLUCIONADO

Los algoritmos pueden representarse de tres formas:

FormaDescripciónUso hoy en industria
Lenguaje natural Descripción en palabras ✅ Documentación, requerimientos, historias de usuario
Diagrama de flujo Representación visual con figuras 🟡 Útil para aprender; en industria se usa BPMN 2.0 o UML Activity Diagrams
Pseudocódigo Notación estructurada legible por humanos ✅ Diseño previo a codificar; estándar en entrevistas técnicas, diseño de sistemas
⚠️

Nota sobre diagramas de flujo: Los diagramas con las figuras clásicas de Cairó (óvalos, rombos, rectángulos) son la herramienta de APRENDIZAJE de este curso. En empresas modernas, los analistas usan herramientas como Lucidchart, Miro o draw.io con notación BPMN. El CONCEPTO es el mismo; la herramienta evoluciona.

5. Símbolos Básicos del Diagrama de Flujo

SímboloFiguraUso
Inicio / FinÓvalo redondeado ⬭Delimita el algoritmo
ProcesoRectángulo □Instrucción o cálculo
DecisiónRombo ◇Condición: bifurca en SÍ/NO
Entrada/SalidaParalelogramo ▱LEER datos / ESCRIBIR resultados
ConectorCírculo ○Conecta partes del diagrama

Cairó
ALGORITMO PromedioCalificaciones VARIABLES nota1 : REAL nota2 : REAL nota3 : REAL nota4 : REAL nota5 : REAL promedio : REAL INICIO ESCRIBIR "=== CALCULADORA DE PROMEDIO ===" ESCRIBIR "Ingrese la nota 1:" LEER nota1 ESCRIBIR "Ingrese la nota 2:" LEER nota2 ESCRIBIR "Ingrese la nota 3:" LEER nota3 ESCRIBIR "Ingrese la nota 4:" LEER nota4 ESCRIBIR "Ingrese la nota 5:" LEER nota5 promedio = (nota1 + nota2 + nota3 + nota4 + nota5) / 5 ESCRIBIR "El promedio es: " + promedio FIN
🔍 Ejecuta para calcular tu promedio...

HTI-B Reto: Calcula la tasa de retención de NeuroBiz (847 asistidas, 112 fallidas).

Cairó
ALGORITMO TasaRetencionNeuroBiz VARIABLES sesiones_asistidas : ENTERO sesiones_fallidas : ENTERO total_sesiones : ENTERO tasa_retencion : REAL INICIO ESCRIBIR "=== NEUROBIZ S.A.S. — TASA DE RETENCION ===" ESCRIBIR "Cuantas sesiones fueron asistidas?" LEER sesiones_asistidas ESCRIBIR "Cuantas sesiones fueron fallidas?" LEER sesiones_fallidas total_sesiones = sesiones_asistidas + sesiones_fallidas tasa_retencion = (sesiones_asistidas / total_sesiones) * 100 ESCRIBIR "Total de sesiones: " + total_sesiones ESCRIBIR "Tasa de retencion: " + tasa_retencion + "%" FIN
