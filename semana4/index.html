1. Condicional doble SI-SINO-FSI 🟢 VIGENTE

La estructura SI-SINO-FSI se usa cuando el algoritmo debe escoger entre dos caminos. Si la condición es verdadera, ejecuta el bloque ENTONCES; si es falsa, ejecuta el bloque SINO.

📖

¿Qué significan las siglas de esta semana?
SI = "Si"  ·  ENTONCES = "entonces ejecuta esto"  ·  SINO = "Si No" (el camino cuando la condición es falsa)  ·  FSI = "Fin Si" (cierra todo el bloque condicional)

DD = "Dependiendo De" (abre la decisión múltiple)  ·  CASO = caso específico a evaluar  ·  FDD = "Fin Dependiendo De" (cierra la estructura)

🐍 Equivalentes en Python:
SI-SINO-FSI → if condición: ... else:
DD-FDD (múltiples casos) → if/elif/else o match/case (Python 3.10+)

🚦

Analogía de vida real: SI el semáforo está en rojo ENTONCES frenar; SINO seguir. Cuando solo hay dos posibles reacciones, esta estructura es la más clara.

2. Decisión múltiple DD-FDD 🟡 EVOLUCIONADO

Cuando existen tres o más casos mutuamente excluyentes, la notación Cairó usa DD ... CASO ... FDD. En la industria moderna suele representarse con if / elif / else o con match / case en Python 3.10+.

EstructuraCuándo usarlaEjemplo NeuroBiz
SI-SINO-FSIDos rutasAprobado / revisar
DD-FDDTres o más rutasVerde / amarillo / rojo
SI anidadoCuando una segunda decisión depende de la primeraPaciente en riesgo y además terapia crítica

3. Condicionales anidadas 🟢 VIGENTE

Una condición anidada aparece cuando un SI está dentro de otro SI. Sirve para refinar decisiones, pero si se abusa de ella el algoritmo se vuelve difícil de leer.

⚠️

Alerta de diseño: si aparecen demasiados SI anidados y los casos son categorías claras, probablemente conviene un DD-FDD o un if/elif/else.

4. Límites y condiciones de frontera 🟢 VIGENTE

Las clasificaciones dependen de definir bien los límites: ¿qué pasa exactamente con 80? ¿Y con 50? Un buen algoritmo evita solapamientos y huecos entre casos.

5. Conexión con Python e industria 🟡 EVOLUCIONADO

El DD-FDD es útil didácticamente, pero en entornos modernos la industria prefiere if/elif/else o match/case. El concepto sigue siendo totalmente vigente.

📚

Nota histórica: muchos libros antiguos, como Deitel en Java, usan switch-case como referencia principal. 🔴 HISTÓRICO Hoy ese ejemplo sirve para contexto, pero el curso conectará con Python en semanas posteriores.

📝 Pseudocódigo — ClasificarPacienteSimple
ALGORITMO ClasificarPacienteSimple
  VARIABLES
    tasa_retencion: REAL
    clasificacion: CADENA
  INICIO
    LEER tasa_retencion
    SI tasa_retencion >= 80.0 ENTONCES
      clasificacion ← "VERDE: Buen progreso"
    SINO
      clasificacion ← "ATENCIÓN: Revisar plan terapéutico"
    FSI
    ESCRIBIR "Estado del paciente: ", clasificacion
  FIN
📝 Pseudocódigo — ClasificarRiesgoNeuroBiz
ALGORITMO ClasificarRiesgoNeuroBiz
  VARIABLES
    tasa_retencion: REAL
  INICIO
    LEER tasa_retencion
    DD tasa_retencion
      CASO >= 80.0: ESCRIBIR "🟢 VERDE — Retención excelente"
      CASO >= 50.0: ESCRIBIR "🟡 AMARILLO — Retención en riesgo"
      CASO OTRO:   ESCRIBIR "🔴 ROJO — Intervención urgente"
    FDD
  FIN

📏

Regla de frontera: antes de dar por bueno un DD-FDD, prueba exactamente los valores límite. Si 80 y 50 no están claros, la clasificación del negocio será inconsistente.

  • 85 valida el caso verde.
  • 65 valida el caso amarillo.
  • 45 valida el caso rojo.
  • 80 y 50 validan fronteras.

Elegir bien la estructura también es una decisión de diseño: claridad primero, complejidad después.

Cuando el estudiante justifica por qué escogió SI-SINO, DD-FDD o anidación, ya está pensando como analista y no solo como copiador de sintaxis.

Ese criterio será clave cuando el curso pase a Python.

⚡ Quiz Semana 4 — Condicional doble, múltiple y anidada

1. En SI-SINO-FSI, si la condición es FALSA se ejecuta:

  1. Ambos bloques
  2. Ningún bloque
  3. El bloque SINO
  4. Se repite la condición
✅ c — Cuando la condición es falsa, corre el bloque SINO.

2. DD-FDD en Python moderno (3.10+) equivale a:

  1. match/case
  2. while
  3. for
  4. import
✅ a — El paralelo más cercano en Python moderno es match/case.

3. ¿En qué casos usar DD-FDD en lugar de SI anidados?

  1. Solo cuando hay 2 casos
  2. Cuando hay 3 o más casos mutuamente excluyentes
  3. Nunca, son lo mismo
  4. Solo en Python
✅ b — DD-FDD organiza mejor 3 o más categorías excluyentes.

4. En NeuroBiz, ¿qué estructura usarías para clasificar tasa ≥80, 50-79 y <50?

  1. MQ-FMQ
  2. HH-FHH
  3. PARA-FPARA
  4. DD-FDD
✅ d — Son tres rutas de clasificación, por eso DD-FDD es la opción natural.

5. La notación Cairó DD-FDD tiene relevancia 🟡 porque:

  1. La industria usa if/elif/else o match/case en Python/JavaScript
  2. Ya no existe en ningún contexto
  3. Es solo para Java
  4. Es más avanzada que Python
✅ a — El concepto sigue vigente, aunque la forma de escribirlo cambie según el lenguaje.
