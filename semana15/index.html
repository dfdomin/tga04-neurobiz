Técnicas de debugging: leer el traceback, insertar print() estratégicos, validar entradas y usar try/except para capturar errores recuperables.

🐍 Python — try/except básico
# Manejo básico de errores
try:
    asistidas = int(input("Sesiones asistidas: "))
    fallidas = int(input("Sesiones fallidas: "))
    total = asistidas + fallidas
    if total == 0:
        raise ValueError("No hay sesiones registradas")
    tasa = (asistidas / total) * 100
    print(f"Tasa: {tasa:.1f}%")
except ValueError as e:
    print(f"Error de valor: {e}")
except ZeroDivisionError:
    print("Error: división por cero")

BUG 1 — NameError
Código buggy
# BUG 1: NameError
def calcular_tasa(asistidas, fallidas):
    total = asistidas + fallidas
    return (asistidas / total) * 100
resultado = calcular_tasa(18, 2)
print(resultadoo)

Pista: el cálculo funciona; el error está en el nombre usado al imprimir.

BUG 2 — ZeroDivisionError
Código buggy
# BUG 2: ZeroDivisionError
def tasa_con_error(asistidas, fallidas):
    total = asistidas + fallidas
    return asistidas / total * 100

Pista: falta validar total == 0 antes de dividir.

BUG 3 — TypeError
Código buggy
# BUG 3: TypeError
sesiones = "18"
fallidas = 2
print(sesiones + fallidas)

Pista: no puedes sumar directamente texto y entero.

BUG 4 — Error lógico
Código buggy
# BUG 4: Error lógico
def clasificar(tasa):
    if tasa > 80:
        return "VERDE"
    elif tasa > 50:
        return "AMARILLO"
    else:
        return "ROJO"

Pista: revisa los bordes exactos 80 y 50.

BUG 5 — IndentationError
Código buggy
# BUG 5: IndentationError
def reporte(pacientes):
for p in pacientes:
    print(p)

Pista: el cuerpo de la función debe ir indentado debajo de def.

1. NameError en Python ocurre cuando:

  1. Se usa una variable no definida
  2. Hay un error de tipo
  3. División por cero
  4. Error de indentación
✅ a — El intérprete no reconoce el nombre porque no existe o está mal escrito.

2. try/except sirve para:

  1. Manejar errores en tiempo de ejecución
  2. Evitar SyntaxErrors
  3. Mejorar velocidad
  4. Comentar código
✅ a — Permite capturar excepciones y responder de forma controlada.

3. print("sesiones" + 18) daría:

  1. TypeError
  2. NameError
  3. SyntaxError
  4. ValueError
✅ a — No se puede concatenar directamente una cadena con un entero.

4. El traceback de Python indica:

  1. El archivo, línea y tipo de error
  2. Solo el tipo de error
  3. Solo la línea
  4. Ningún dato útil
✅ a — El traceback es el mapa de dónde falló el programa.

5. Debugging con print() es efectivo cuando:

  1. Quieres ver el valor de variables en puntos específicos
  2. Hay errores de sintaxis
  3. El código no se ejecuta
  4. Hay problemas de red
✅ a — print debugging permite observar el estado interno del programa.
