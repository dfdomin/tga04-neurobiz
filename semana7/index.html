📚 Marco teórico

Cuerpo al menos una vez 🟢 VIGENTE

HH-FHH se usa cuando la interacción debe suceder antes de evaluar la condición. Por eso es ideal para formularios, validaciones y menús.

⏰
Analogía vida real: una alarma suena y luego pregunta si quieres posponerla.
🏢
Analogía empresarial: un formulario de préstamo se muestra al menos una vez antes de verificar si el cliente firmó.

Comparación con MQ-FMQ

EstructuraPreguntaEjemplo

HH-FHH¿Debo repetir después de intentar?Validar cédula
🐍
Python no tiene do-while nativo; la lógica se implementa con while True y break.
⚠️
Si no diseñas una salida clara, el menú se queda atrapado en un bucle.

📝 Pseudocódigo y equivalencias

📝 HH-FHH — Validar cédula
ALGORITMO ValidarCedulaNeuroBiz
  VARIABLES
    cedula: ENTERO
    intentos: ENTERO
  INICIO
    intentos ← 0
    HH
      ESCRIBIR "Ingrese número de cédula del paciente:"
      LEER cedula
      intentos ← intentos + 1
      SI cedula <= 0 ENTONCES
        ESCRIBIR "⚠️ Cédula inválida. Intente de nuevo."
      FSI
    FHH MIENTRAS cedula <= 0
    ESCRIBIR "✅ Cédula registrada: ", cedula
    ESCRIBIR "Intentos requeridos: ", intentos
  FIN
📝 HH-FHH — Menú clásico
ALGORITMO MenuNeuroBiz
  VARIABLES
    opcion: ENTERO
  INICIO
    HH
      ESCRIBIR "=== SISTEMA NEUROBIZ ==="
      ESCRIBIR "1. Registrar sesión"
      ESCRIBIR "2. Ver estadísticas"
      ESCRIBIR "3. Generar reporte"
      ESCRIBIR "0. Salir"
      LEER opcion
      DD opcion
        CASO 1: ESCRIBIR "Registrando sesión..."
        CASO 2: ESCRIBIR "Calculando estadísticas..."
        CASO 3: ESCRIBIR "Generando reporte..."
        CASO 0: ESCRIBIR "Hasta luego."
        CASO OTRO: ESCRIBIR "Opción inválida"
      FDD
    FHH MIENTRAS opcion <> 0
  FIN
🐍 Equivalente en Python
while True:
    cedula = int(input("Ingrese cédula: "))
    if cedula > 0:
        break
    print("Cédula inválida")
print(f"Cédula registrada: {cedula}")
